Software Development Lyfecycle
- a process to follow when developing sofware
- SDLC defines the phases necessary for a project from its initial
    creation to deployment and maintenance

SDLC Phases
- Requirements Phase
  - Phase where planing for the upcoming project happens
  - What needs to be developed?
  - what sort of functionality are we envisioning?
  - What is the project schedule going to look like?
  - Business Analysts + Senior team members
- Analysis Phase
  - Gather together documents for the new system
  - Resolve any ambiguities and inconsistencies in the requirements
  - A software requirement specification document is created
    - Software and hardware requirements are formaly defined
  - Bas + Senior developers
- Design Phase
  - Outline technical details for the appiclation
    - Infrastucture
    - Database structure
    - System interfaces
    - User interfaces
  - Pseudo-code, diagrams, wireframe, etc.
  - System architects + Senior developers, UX designers
- Developmet Phase
  - Developers write code and build the application based on the 
      design documents
  - Junior + Senior developers
- Testing Phase
  - Equally and important as the development phase
  - The software is tested to find bugs
  - bugs that are found are fixed
  - The sofware at the end of this phase should meet the requirements
      laid out in the requirements and analysis phase
  - QA members(Testers) + Developers
- Deployment Phase
  - The application is deployed to the dedicated serves for hosting
  - System Administrators + Developers
- Maintanance Phase
  - Ensure that the system is up and functional has high availability
  - Resolve new issues based on the user reports
  - Develepors + System Administrators + Site Relability Engineers +
     Customer support + Business Analysts
